# 岚城联：一季之内

一款关于足球俱乐部经营、承诺与后果的单页叙事游戏。

这次重构把原先“每阶段一道管理选择题”的结构，改成了一个完整赛季的十集故事。玩家扮演足球运营总经理，在信息不完整的情况下处理董事会、现金流、比赛模型、队长合同、看台冠名、换帅、伤病、冬窗与赛季冲刺。

## 当前版本

- 十集完整赛季，从7月上任到次年5月赛季复盘
- 每集两个以上冲突现场
- 每集三条线索，但玩家最多只能追问其中两条
- 每集三个不可逆决定，共30个决定和30段专属余波
- 人物会记住承诺、失信与此前没有解决的事项
- 旧决定会以新闻、合同、身体、付款和人的态度延迟返回
- 财务线同时区分现金流、球队成本限额与球员注册，包含虚构联赛的 1:4 整改规则
- 球迷线包含缴费会员、一人一票、会员大会、选出代表与治理席位
- 换帅后会动态更新主教练身份，未换帅路线保留贺峥并显示对应回声
- 仅显示可确认的财务与合同事实，不显示好感度、竞技属性或“正确答案”式收益
- 关键比赛包含随机性，不用比赛结果反向证明某个经营决定必然正确
- 赛季结束后才显示真实足球案例框架和管理复盘
- 浏览器本地保存进度，可导出完整任期记录 JSON

## 本地运行

页面需要读取 `story-data.json`，请通过本地网页服务打开：

```bash
python3 -m http.server 8173
```

然后访问：

```text
http://127.0.0.1:8173/index.html
```

## 验证

```bash
node --check app.js
node tests/story-smoke.mjs
```

完整性测试会验证十集结构、追问与决定数量、专属余波、页面元素引用，并自动结算三条不同路线上的付款、承诺和人物状态。

## 主要文件

- `story-data.json`：十集剧情、追问、决定、余波、延迟回声和赛后复盘
- `app.js`：场景播放器、状态记忆、到期付款、比赛、存档和结局
- `index.html`：游戏界面
- `styles.css`：响应式视觉系统
- `岚城联_沉浸式剧情线重构方案.md`：本轮重构的叙事与系统设计依据
- `tests/story-smoke.mjs`：三路线完整性测试

## 设计边界

玩家负责合同、预算、组织原则和责任分配，不直接挑选首发或在比赛中点击阵型。比赛表现由此前积累的阵容、体能、凝聚力、教练权威和偶然性共同影响。
