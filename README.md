# 岚城联：一季之内

一款关于足球俱乐部经营、承诺与后果的单页叙事游戏。

这次重构把原先“每阶段一道管理选择题”的结构，改成了一个完整赛季的十集故事。玩家扮演足球运营总经理，在信息不完整的情况下处理董事会、现金流、比赛模型、队长合同、看台冠名、换帅、伤病、冬窗与赛季冲刺。

## 当前版本

- 上任序章从一场共同事件开始：玩家先在雨里扶住旧壁画下的油漆桶，再从维修师傅、旧广播、季票与空柜子中逐步拼出球队历史
- 长对白按同一说话人和信息块合并为每次2—4句的段落节拍；换人、证据揭示与选择前仍保留停顿
- 油漆桶、十七号水杯、队长磁牌、付款单、医疗表和积分榜会跨章节重复出现，让上一选择通过可见物件进入下一场
- 十集均加入“本集行程”，提前显示即将经过的地点与任务，避免人物和场景无预告跳转
- 前九集加入章间人物来信与下一站预告；上一决定会通过人物原话、具体变化和时间推移接到下一章
- 前三集分别采用“最后去见谁”“方案压力测试”“失败录像重放”，不再重复同一套问卷流程
- 九名主要人物都有习惯动作、语言节奏和持续行为状态；内在动机不直接展示，由对话和行动体现
- 正文中的足球术语可就地打开白话解释，另有完整“足球词语”面板
- 手机端常驻“帮助 / 足球词语”入口；正式决定前显示本集目标、硬约束、已知风险和未核实信息
- 第二集把银行现金、已承诺付款和联赛注册成本空间分成两条可复算通道，并明确1:4整改公式与有效期
- 财务侧栏同时显示今日现金与已知付款全部兑现后的前景；到期资金不足会触发融资、紧急出售或延期付款危机
- 经营侧栏公开六维直接战力（进攻、防守、体能、默契、士气、阵容深度）、六项间接竞技支撑、综合战力与真实胜平负概率
- 每次决定后显示专属影响画面：决定前后雷达图、逐项数值变化、财务变化和剧情化伏笔同时出现
- 决定结果先显示承诺、最大收益、最大代价与下一风险；人物反应、雷达图和完整指标可按需展开
- 决定结果把可行动的“管理提醒”和文学性的“散场画面”分开；训练收益另有可随时查看的待兑现/已兑现台账
- 正式决定采用“完整原话复述 → 确认公开/返回比较”两步提交，已有存档时新开任期会明确提示覆盖
- 第二、三集的组织授权立即记录，体能、攻守与默契等竞技变化在后续训练或比赛节点兑现并出具回执
- 旧决定不只以台词回归：跨章节后果会在预定剧情节点再次改变战力、组织或现金，并显示“当时选择—今日兑现”的数值回执
- 前三集的决定会写入因果账本；第三集赛后按具体分钟展示补强、阵容深度、战术选择与偶然事件怎样共同形成比分
- 完整十集均已加入动漫视觉小说画面：21个场景背景、12名角色和36种表情立绘
- 关键冲突支持双人物同场、左右站位、远近景与说话焦点切换
- 背景换场采用较慢的交叉溶解，人物进入、退场和对白淡入降低位移幅度，雨雪和摄影棚带有克制环境动态
- 说话人与镜头焦点彼此独立，记者、委员会或会议叙述不会再被画面人物姓名覆盖
- 桌面端提供“剧情聚焦”模式，可暂时收起经营侧栏并扩大主舞台
- 十集完整赛季，从7月上任到次年5月赛季复盘
- 每集两个以上冲突现场
- 每集三条人物立场；前三集只能在真实时间约束下选择一条，第四至第十集沿用最多核实两条的结构
- 每赛季另有四次“主动了解”；前三集的互动已嵌入现场，第四集起可在事件推进中自行发起
- 每集三个不可逆决定，共30个决定和30段专属余波
- 最终结局同时读取十集的选择倾向、人物是否仍愿意直说、承诺是否失信和期末现金，不由最后一个选项单独决定
- 人物会记住承诺、失信与此前没有解决的事项
- 旧决定会以新闻、合同、身体、付款和人的态度延迟返回
- 财务线同时区分现金流、球队成本限额与球员注册，包含虚构联赛的 1:4 整改规则
- 球迷线包含缴费会员、一人一票、会员大会、选出代表与治理席位
- 换帅后会动态更新主教练身份，未换帅路线保留韩立锋并显示对应回声
- 人物好感仍不显示；竞技属性和财务影响公开，但不把复杂取舍标成“正确答案”
- 关键比赛包含明确标注的随机性，并区分“决定造成的比赛形状”和“比分中的偶然”，不用赛果反向证明某个经营决定必然正确
- 比分、胜平负、标题和事件时间线由同一赛果对象生成；0-0、非零平局、主胜和主负均有自动一致性校验
- 赛季结束后才显示真实足球案例框架和管理复盘
- 浏览器本地保存进度；保存对象含构建/schema、单调 revision、检查点和状态 hash，并拒绝旧标签页静默覆盖较新进度
- 测试可通过 `?save=独立标识` 使用隔离存档命名空间，界面会显示当前构建、存档标识和最近检查点

## 本地运行

页面需要读取 `story-data.json`，请通过本地网页服务打开：

```bash
python3 -m http.server 8173
```

然后访问：

```text
http://127.0.0.1:8173/index.html
```

## GitHub Pages

这个项目不需要服务器、数据库或构建步骤，可以直接部署为 GitHub Pages 静态网页：

1. 把本目录中的文件推送到 GitHub 仓库的默认分支。
2. 在仓库的 **Settings → Pages** 中，把发布来源设为 **Deploy from a branch**。
3. 选择默认分支和根目录 **/(root)**，保存后等待 GitHub 给出网页地址。

`index.html`、剧情 JSON 和图像都使用相对路径，因此部署在 `https://用户名.github.io/仓库名/` 这样的子目录地址时也能直接运行。

## 视觉与剧情分离

- `story-data.json` 仍是剧情、对话、选项和后果的唯一来源。
- `visual-data.json` 保存十集的背景、人物、表情和场景映射，不复制剧情文字。
- 同场人物使用 `supportingCharacter`、`supportingExpression` 与左右位置字段；未配置这些字段的旧场景仍保持单人物播放。
- `assets/visuals/` 保存可替换的图像；替换同路径图片或修改视觉映射，不会改动剧情。
- 视觉配置或图片加载失败时，游戏会自动继续显示原有文字，不阻断游玩。

## 验证

```bash
node --check app.js
node tests/story-smoke.mjs
```

完整性测试会验证十集结构、追问与决定数量、专属余波、页面元素引用，并自动结算三条不同路线上的付款、承诺和人物状态。

## 主要文件

- `story-data.json`：十集剧情、追问、决定、余波、延迟回声和赛后复盘
- `visual-data.json`：完整十集使用的视觉映射，与剧情数据独立
- `assets/visuals/`：动漫背景、透明人物立绘和可再次编辑的生成母版
- `app.js`：场景播放器、多维战力、胜负概率、即时/延迟后果、状态记忆、到期付款、存档和结局
- `index.html`：游戏界面
- `styles.css`：响应式视觉系统
- `岚城联_沉浸式剧情线重构方案.md`：本轮重构的叙事与系统设计依据
- `docs/序章视觉设定档案.md`：人物与场景的原文约束、允许补充和统一画风
- `docs/序章图像生成记录.md`：GPT Image 2 提示词与素材对应关系
- `docs/全赛季视觉设定档案.md`：第二至第十集的原文约束、允许补充与视觉连续性
- `docs/全赛季图像生成记录.md`：后续章节的 GPT Image 2 提示词、处理流程与素材清单
- `docs/galgame章节视觉化_skill草案.md`：本项目采用的章节视觉化工作方法草案
- `docs/核心体验重构蓝图_v1.md`：人物、术语、因果反馈、文本与金钱压力的本轮改造蓝图
- `tests/story-smoke.mjs`：三路线完整性测试

## 设计边界

玩家负责合同、预算、组织原则和责任分配，不直接挑选首发或在比赛中点击阵型。比赛表现由此前积累的阵容、体能、凝聚力、教练权威和偶然性共同影响。
