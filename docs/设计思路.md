# 岚城联：设计思路

版本：0.5
项目形态：学生端静态网页 MVP  
目标玩家：初高中生，高中生为主

## 1. 这个游戏要教什么

这个游戏不是为了让学生学习“怎么买最强球员”，而是让学生体验足球俱乐部作为一个复杂组织的经营逻辑。

核心教学目标有三点：

1. 资源有限时，经营就是取舍。钱、时间、球员、球迷、文化和成绩不能同时拉满。
2. 一个选择通常不只影响一个结果。买人可能提高实力，也可能破坏薪资结构；保守经营可能稳定文化，也可能错过短期成绩。
3. 足球有随机性，但随机性不是完全无原因的。伤病、更衣室、舆论、董事会压力等风险，经常来自此前积累的经营状态。

因此，游戏的重点不是让玩家追求一个唯一最优解，而是让每个玩家经营出一支“有来路、有代价、有性格”的球队。

## 2. 为什么把系统分成三层

游戏把球队拆成三层：

```text
财务硬约束
经营状态
竞技状态
```

### 财务硬约束

财务不做成“财务健康分”，也不直接进入战力公式。

它的作用是限制玩家能不能选某些方案。例如现金不足时，昂贵引援、医疗投入、教练组强化等选项会直接不可选。

这样设计的原因是：真实俱乐部的财务压力首先表现为“你没有资格做某些选择”，而不是一个抽象分数。

### 经营状态

经营状态包括青训体系、球探网络、球迷信任、社区连接、管理层耐心、媒体关系、俱乐部文化、队内权力平衡、商业吸引力。

这些变量本身不等于场上实力。它们更像俱乐部生产竞争力的环境。

例如：

```text
青训体系高 → 后续可能出现更便宜的年轻球员选项
球迷信任高 → 主场气势和心理稳定性更容易提升
队内权力平衡低 → 更容易触发更衣室矛盾
媒体关系差 → 小矛盾更容易被放大
```

### 竞技状态

竞技状态才是比赛算法直接读取的内容，包括球员水平、阵容厚度、战术熟练度、体能健康、更衣室凝聚力、心理稳定性、教练临场能力、主场气势。

玩家不是球员，也不是主教练，所以玩家不能直接“点一下就让球队变强”。玩家只能通过预算、引援、续约、赞助、医疗、更衣室处理等经营动作，间接改变竞技状态。

## 3. 为什么采用赛季时间线

游戏采用 10 个赛季阶段：

```text
上任定位
赛季预算
球员来源
核心问题
教练路线
赞助与社区
更衣室
伤病与医疗
第一场主场
第一场客场
```

这样设计有两个好处：

1. 所有学生共享同一条赛季框架，便于课堂讨论和复盘。
2. 每个阶段又会根据前面选择进入不同事件，保证每个学生的球队经历不完全相同。

它不是纯粹的线性剧情，也不是无限分叉的大树，而是：

```text
共同阶段 + 条件事件池 + 状态传导
```

这个结构能避免分支爆炸，同时保留“我自己的球队是被我自己经营出来的”这种体验。

## 4. 为什么不是每个选择固定跳转

如果做成传统树状剧情：

```text
选 A → 跳到节点 2
选 B → 跳到节点 3
```

很快会出现分支爆炸。10 个阶段后，剧情数量会变得难以维护。

现在的做法是让每个阶段都有一个事件池：

```text
第 4 阶段：核心问题
├─ 如果签过成名老将 → 新援高薪刺激核心不满
├─ 否则如果俱乐部文化高 → 核心支持重建
└─ 否则 → 默认核心续约危机
```

这种结构的重点不是“跳到哪里”，而是“当前球队状态使你遇到什么问题”。

同一个选择在不同球队里也可能产生不同后果，因为后续触发条件不同。比如同样是买球星，如果此前队内权力平衡已经很差，就更容易在更衣室阶段爆雷。

## 5. 为什么开局要先走一段序章剧情

早期版本里，玩家一上来就看到很多数值和资料卡，但这会让游戏像阅读说明书。玩家虽然知道了信息，却没有真正进入角色。

现在开局改成 6 段序章剧情：

```text
旧看台下的新任经理
主席办公室
第一场发布会
一线队训练场
核心球员的沉默
青训基地与财务室
```

玩家不再先读一份报告，而是通过主席、记者、主教练、核心球员、青训主管和财务主管这些角色，逐步理解自己接手的处境。

这些序章场景不改变数值。它们的作用是让第一个选择有叙事重量：玩家不是在答题，而是在上任第一天后必须给出路线。

## 6. 为什么数值要有相对评价

只显示 `青训体系 50` 或 `体能健康 43`，玩家很难判断意义。

现在数值被解释成区间：

```text
0-34：危机
35-44：偏低
45-55：中游
56-65：良好
66-75：强项
76-100：顶级
```

这让学生明白：50 不是“满分的一半所以很差”，而是“联赛中游平均线附近”。低于 45 开始有明显隐患，高于 60 才算稳定优势。

## 7. 为什么每个选项要解释钱的量级

朋友试玩时提出一个关键问题：这些钱为什么是这个数？

所以现在每个选项会显示财务口径，例如：

```text
立即支出约占开局现金 29%
属于中游队的一线队级大额操作
工资承诺增加会成为后续续约和更衣室比较的参照
未来第 8 阶段还要支付设施尾款
```

这些说明不是会计模拟，而是帮助玩家理解金钱在这个虚构联赛里的尺度。

开局银行余额是 4200 万，其中 600 万为不能挪用的青训专款，因此界面显示的可动用现金是 3600 万。学生可以据此估算：一笔 1200 万支出会用掉三分之一的可动用现金，不是普通的“购买按钮”。

金额解释不能只说“大约占开局现金多少”，还必须说明钱从哪里来、具体买了什么。例如：

```text
现金 +800 → 董事会以联赛前四承诺为条件追加的临时预算
现金 -550 → 运动医学顾问、两名康复师、设备租赁和赛季检测服务
现金 +1400 → 企业购买球衣广告、看台命名和球员形象使用权
现金 +1600 → 另一家俱乐部支付的林骁转会费
```

如果一个行动本来不应花钱，就不为了制造数值波动而强行收费。0.5 版本因此取消了队长内部会议的 80 万费用，也取消了基础客场差旅已经支付后再次扣除的 80 万。

## 8. 为什么选项必须是具体行动

玩家不能只在“稳健、冲刺、文化、平衡”这些抽象词之间选择。每个选项现在都采用四层表达：

```text
选项名称：一句话概括要签的合同或要执行的安排
你会做什么：具体人物、岗位、名单、期限和条款
金额怎么来：资金来源或报价构成
代价与收益：谁获益、谁承担风险、后续什么会改变
```

例如第一阶段不再让玩家选择“明确冲击成绩”，而是选择“接受董事会追加 800 万，承诺冲击联赛前四”。这让玩家知道自己不是表达态度，而是在签署一份附带问责条件的会议纪要。

## 9. 为什么选项影响要多维

足球经营决策很少只影响一个指标。

例如“签下成名老将前锋”不只是：

```text
球员水平 +8
现金 -1100
```

它还会带来：

```text
商业吸引力上升
球迷信任上升
工资承诺上升
队内权力平衡下降
更衣室凝聚力下降
后续核心续约危机更容易变形
```

这种多维影响是游戏的核心：学生要看到一个选择的附带后果，而不是只看眼前收益。

## 10. 为什么选择后要有反馈剧情

早期版本里，玩家点完选项会直接进入下一阶段，体验上很像问卷。

现在每次选择后都会先出现一段“选择后的回声”。反馈先确认玩家实际签署或安排了什么，再根据选择造成的影响继续生成剧情，例如：

```text
花了大钱 → 财务主管提醒后续空间变窄
球迷信任上升 → 球迷论坛开始转发你的决定
队内权力失衡 → 更衣室门关得更快
核心离队 → 孩子举着旧海报站在训练基地门口
体能风险上升 → 队医报告出现更多红色标记
```

这样做是为了让玩家感到选择真的进入了球队生活，而不是只改变了几个数字。

## 11. 为什么随机事件要有因果感

游戏里有随机比赛结果，也有状态触发事件。

但设计原则是：随机不能让玩家觉得“系统乱扣分”。它应该像足球世界里的风险显形。

例如：

```text
体能健康低 → 客场前夜更容易进入体能透支分支
阵容厚度高 → 伤病阶段可以主动轮换
赞助走大型商业路线 → 主场可能出现商业化冷场
权力平衡低 → 替补更容易公开抱怨
```

也就是说，随机性负责制造不确定性，状态系统负责解释为什么这种不确定性更容易发生在这支球队身上。

## 12. 比赛算法的定位

当前 MVP 只模拟两场比赛：

```text
第一场主场
第一场客场
```

比赛算法直接读取竞技状态，而不是读取经营状态或财务状态。

这符合游戏的核心设定：财务、青训、球迷、媒体不是场上战力本身，而是通过引援、心理、体能、战术、阵容厚度等中间环节影响比赛。

比赛结果包括胜、平、负。结果又会反过来改变球迷信任、管理层耐心、媒体关系、心理稳定性和更衣室凝聚力。

## 13. 为什么结尾要先是尾声剧情

学生最终仍然需要导出数据码，供教师端或后续联赛模拟使用。

但数据码不应该成为玩家的情感结尾。现在结尾先展示一段尾声剧情：

```text
第一次客场结束后的晚上，经理回到办公室。
桌上有主场首战剪报、董事会会议提醒、训练基地通行证。
剧情会根据核心去留、青训路线、赞助路线、胜负结果等内容回顾玩家选择。
```

然后才显示 JSON 数据码。

这样结尾会和开头的上任第一天呼应：玩家作为经理接手了一支球队，也亲手把它带到了一个新的处境。

## 14. 当前 MVP 的边界

当前版本是学生端单队经营 MVP，不是完整联赛系统。

已经实现：

```text
玩家填写经理人与球队名
6 段开局序章
10 个赛季阶段
条件事件池
财务硬门槛
经营状态与竞技状态
每次选择后的反馈剧情
两场比赛模拟
赛后报道
动态尾声剧情
球队数据码导出
剧情分支树文档
```

尚未实现：

```text
15 支学生球队双循环联赛
教师端批量导入数据
积分榜与赛程表
更复杂的随机事件池
教师隐藏资料卡展示界面
完整两小时版本剧情
```

## 14. 后续扩展方向

### 教师端联赛模拟器

学生完成游戏后导出球队数据码。教师端可以导入 15 支球队，生成双循环联赛：

```text
15 支球队
每两队主客场各打一场
总计 210 场
胜 3 分，平 1 分，负 0 分
```

比赛算法可以继续读取每支球队的竞技状态，并加入伤病、天气、主客场、士气等随机因素。

### 不同初始球队

后续可以让学生开局抽到不同类型的俱乐部：

```text
青训强但没钱
资本雄厚但球迷疏离
老牌豪门但工资结构失衡
小城球队但社区连接极强
新老板入主但管理层耐心很低
```

这样同一个选择在不同初始条件下会产生更明显的差异。

### 教师资料卡

每个节点背后都可以绑定真实足球案例，但不直接展示给学生。

课后复盘时，教师可以讲：

```text
核心球星续约：梅西与巴萨、C 罗与皇马
会员制与财政：巴萨、皇马、拜仁
资本路线：曼城、巴黎、切尔西
球迷抗议：曼联与格雷泽
管理混乱：AC 米兰
```

这样游戏先让学生体验问题，课堂再把体验连接到真实足球史。

## 15. 一句话总结

这个游戏的设计核心是：

```text
让学生在有限资源、组织压力、足球随机性和长期发展之间做取舍，
并通过每一次选择看到复杂组织的连锁后果。
```

它不是要证明哪一种经营路线绝对正确，而是让学生理解：足球俱乐部不是 90 分钟比赛机器，而是一套由钱、权力、球员、球迷、文化、媒体和规则共同构成的组织。
