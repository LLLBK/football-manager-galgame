## 后期处理

1. 背景母版保存在 `assets/visuals/source/`，转换为质量 84 的 WebP 后放入 `assets/visuals/backgrounds/`。
2. 人物母版先移除纯洋红背景，再按三等分裁切；每格导出独立透明 WebP 到 `assets/visuals/characters/<角色ID>/`。
3. 全部 36 张人物立绘均检查透明通道与透明角像素；背景和人物路径由自动测试逐一验证。
4. 最终场景、人物、表情与轻动画映射只保存在 `visual-data.json`；图像调整不要求改动 `story-data.json`。

## 全视觉层缓存修订（2026-07-19）

实机检查发现，上一轮只为六名重新生成人物启用了 `*-v2.webp`，其余六名人物和21张背景仍沿用旧 URL。即使 `visual-data.json` 的角色与场景映射已经正确，浏览器仍可能从缓存返回这些 URL 过去对应的错误图片，造成文字已经切换到许青禾、方雯或韩立锋，画面却继续显示周绍庭和董事会议室。

本轮将所有实际使用的背景与人物立绘统一切换到 `*-v2.webp` 路径，并把视觉配置版本升级为 2。旧文件继续保留作回退，不覆盖原始素材。自动测试同时新增三项检查：所有运行时素材必须使用新版路径、人物路径必须属于自己的角色目录、不同视觉条目不得引用内容完全相同的图片。
